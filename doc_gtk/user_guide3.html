Configuration

Most of Privtool can be configured using the Properties window which is accessed via the Edit->Properties menu item. This is a notebook style window with the various aspects of Privtool having their own page of options.

Any changes to the configuration will only be saved upon pressing the OK or Apply buttons.

Some of these windows contain a grid area into which text can be entered. There will always be one extra row for entering new data, if you start typing in the last row another row will automatically be created. Also enpty rows are ignored.

Mailer

The first page presents general mailer configuration options. These are:

Mail Spool File The file where incoming mail is delivered. If this field is left empty then the spool file defaults to either /var/spool/mail/<username> or to a compiled in default.
Check Interval How often (in seconds) privtool will check for new mail.
Mail File Directory The directory where the mail folders reside.
Print Command The command which will have the mail item piped to it for printing.
Organization The text that will be placed in the Organizaation mail header field.
Folder Combo Items Commonly used folders can be placed in this area which will then be used to populate the folder combo on the toolbar.

Compose

These configuration options pertain to composing email.

Reply-To Address The address to be used instead of the From address when people reply to your email.
Domain The domain which get's appended to your username to build the From address. This only needs to be set if the system doesn't get it right automatically
Indent Prefix The string to prepend onto each line of an included mail for quoting purposes.
Wrap Column The column at which text should be wrapped. Setting this to 0 will turn wrapping off, otherwise the text will be wrapped at the space before the specified column.
Log Folder Name The name of a folder where outgoing mail will be saved. (Outgoing mail is saved when the Log checkbox is checked in the compose window.)
Signature File This specifies the name of the signature file to use. If specified then the file will be appended to every message that you send. The filename is assumed to be relative to the user's home directory unless it begins with a '/'.
Alternatively, if the filename begins with a '|' then the rest of the filename is interpreted as a program to run which will produce the "signature" on its standard output.
Options
Sign Outgoing Mail The default setting to specify whether sent mail should be digitally signed.
Encrypt Outgoing Mail The default setting to specify whether sent mail should be digitally encrypted.
Log Outgoing Mail The default setting to specify whether sent mail should be saved.
Log As Ascii The default setting to specify whether sent mail which is encrypted should be saved as plain text.
Beep on Bad Signature The default setting to specify whether privtool should beep if it can't validate a digital signature.
Reply Address Method How to handle replying to mail that has both From: & Reply-To: headers. This can be one of:
Use From: - Always use the From: header address.
Use Reply-To: - Always use the Reply-To: header address.
Ask - Show an alert asking which address to use.

Aliases

This window contains personal mail address aliases. It is not a full featured address book, there is simply a column for the alias and a column for the full email address.

Mail Headers

This window contains a number of check boxes for selecting which headers should be displayed by default. Of course all headers can be displayed by using the View->Full Header menu item.

Security

These properties affect the behaviour of the PGP & security aspects of Privtool.

PGP Key Translations This allows you to specify a mapping from a mail address to a pgp key. You MUST specify such a mapping for your own id in order to make signatures and encrypted logging work.
Security Level Privtool supports four security levels:
Level 1 - read passphrase from PGPPASS Level 2 - ask for passphrase when first required and store it in memory until it is cleared either by time delay or user selection. Level 3 - ask for passphrase every time it is needed and destroy it as soon as possible. Level 4 - currently identical to level 3, in the final release it will delete the decrypted text of each message when you close the display window.
Test Interval If retrieveinterval is set to specify the time periods at which Privtool will check for new mail in your mailbox, then you can also set testinterval to a time in seconds. Each time Privtool checks for mail, it will also check that a message has been displayed in the previous 'testinterval' seconds. If not, then it will destroy your passphrase, regardless of the security level you are running at.
PGP Pseudonyms This allows you to specify an additional identity that you wish to use when using Privtool. The list of identities will appear on the Nym menu, and can be selected from the menu. After the selection, if you use the compose window then messages will be sent as if they came from that pseudonym (e.g. signed with the nym's key). If you use any pseudonym other than your real user id, then the compose window will default to sending messages through remailers.
Kill Mail Matching This allows you to set up automatically ignored email. If the first column contains a 'U' then the second column should specify a user name to ignore. If the first column contains a 'S' then the second column is a subject to ignore.

Ignored mail is actually just automatically marked as deleted and can be subsequantly undeleted if so desired.

Default Pseudonym This allows you to specify the default pseudonym to use, otherwise Privtool will default to your real identity.
